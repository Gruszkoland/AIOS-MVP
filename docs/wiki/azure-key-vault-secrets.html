Azure Key Vault Secrets — Szybki Start

1. CLI (konsola)

  1. Logowanie do Azure: 
    az login
  2. Wybierz subskrypcję: 
    az account set --subscription "NAZWA_LUB_ID_SUBSKRYPCJI"
  3. Utwórz Key Vault: 
    az keyvault create --name NAZWA_VAULT --resource-group NAZWA_RG --location westeurope
  4. Dodaj sekret: 
    az keyvault secret set --vault-name NAZWA_VAULT --name "DbPassword" --value "SuperTajneHaslo123"
  5. Odczytaj sekret: 
    az keyvault secret show --vault-name NAZWA_VAULT --name "DbPassword" --query value -o tsv
  6. Lista wszystkich sekretów: 
    az keyvault secret list --vault-name NAZWA_VAULT -o table

2. Python (SDK)

Instalacja: 
pip install azure-identity azure-keyvault-secrets
Przykład kodu: 
from azure.identity import DefaultAzureCredential
from azure.keyvault.secrets import SecretClient
import os

vault_url = "https://twoj-vault.vault.azure.net"  # lub z os.environ["KEY_VAULT_URL"]
credential = DefaultAzureCredential()
client = SecretClient(vault_url=vault_url, credential=credential)

# Ustawianie sekretu
client.set_secret("ApiToken", "abc-123")

# Pobieranie sekretu
secret = client.get_secret("ApiToken")
print(secret.value)
Uwaga: Musisz być zalogowany przez az login w tym samym środowisku.

3. Najlepsze praktyki

  • Nie trzymaj haseł w plikach .env na produkcji.
  • Używaj RBAC i Managed Identity w chmurze.
  • Ogranicz dostęp do Key Vault tylko do niezbędnych ról.
  • Włącz rotację i audytowanie sekretów.
