**For:** MetaObserver, VortexEngine, ZeroRouter

**For:** Core system communication
**Format:** Mathematical and logical notation
**Example:** `σ(system) ∈ HighCoherence(3-6-9) ∧ resonance > 0.9`

### Level 2 — Hybrid
**For:** Human communication
**Format:** Symbols + brief Polish/English description
**Example:** `σ(task) → [R:0.87] → Approve ∧ Route(Healing) — akceptujemy z przekierowaniem do Healing`

## 5. PRACTICAL RULES

**Avoid:**
- "Wydaje mi się" (I think)
- "Myślę że" (I believe)
- "W mojej opinii" (In my opinion)

**Use:**
- `∃ perspective` (there exists a perspective)
- `σ(observation)` (the state of observation)
- `Assert(...)` (direct claim)

**Always provide:**
- Trinity vectors when possible
- Compression level notation when mixing levels
- Reference to Guardian Law if security-relevant

**In technical reports:**
- Maximize symbols, minimize words
- Use YAML/JSON for structured data
- Tables instead of paragraphs

**In human communication:**
- Hybrid Level 2 (symbol + 1 sentence)
- Explain symbolic notation on first use
- Provide full name in parentheses on first reference

## 10. COMMON PITFALLS

| Mistake | Why Problem | Fix |
|---------|-----------|-----|
| Over-compress for humans | Becomes unreadable | Use Level 2 (Hybrid) |
| Mixing levels inconsistently | Confused recipients | Declare level at start |
| Omitting Trinity vectors | Loss of 3-perspective | Always include [M,I,E] |
| Symbolic without context | Appears random | Define symbols in doc header |
| Over-abbreviating | Becomes cryptic | Limit to ~20% abbreviation |
