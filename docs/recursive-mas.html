Architektura

Czym jest RecursiveMAS?

System multi-agentowy traktujący cały ekosystem LLM jako jedną rekurencyjną obliczeniowość.

RecursiveLink

Wymiana stanów latentnych

Lekkie moduły łączące agentów — iteracyjna wymiana i ewolucja reprezentacji wewnętrznych między rundami rekurencji.

Heterogeniczność

Różne modele, jeden system

Agenty oparte o różne rodziny LLM (Qwen, Llama, Gemma, DeepSeek) współpracują przez wspólny interfejs RecursiveLink.

4 Style

Sequential · Mixture · Distillation · Deliberation

Każdy styl modeluje inny wzorzec kolaboracji — od sekwencyjnego wnioskowania po dynamiczne wywołania narzędzi.

MCP Server

Natywna integracja z adrion-369

Uruchamiany jako kontener Docker w warstwie MCP — dostępny dla Oracle, Genesis i pozostałych agentów systemu.

Konfiguracja

Style kolaboracji

Dobierz styl do zadania — im cięższy, tym lepsze wyniki przy większym zapotrzebowaniu na VRAM.

Styl Agenty VRAM Zastosowanie
sequential_light Planner 1.7B · Critic 1B · Solver 1.5B ~8 GB Szybkie pipeline, standardowe zadania
sequential_scaled Planner 4B · Critic 3B · Solver 4B ~24 GB Złożone rozumowanie, wysokiej jakości output
mixture Domain agents + Summarizer 2B ~20 GB Wielodomenowe analizy (math + code + science)
distillation Expert 9B + Learner 4B ~28 GB Optymalizacja inference, transfer wiedzy
deliberation Reflector 4B + ToolCaller 4B ~16 GB Integracja narzędzi (Tavily), refleksja

Jak to działa w systemie?

RecursiveMAS działa jako niezależny kontener w warstwie MCP, dostępny przez HTTP dla wszystkich agentów.

Oracle MCP
→
MCP Router
→
RecursiveMAS :8095
POST /tools/recursive_mas_infer · GET /tools/recursive_mas_styles · GET /tools/recursive_mas_status
⚡
Wymagania Hetzner (GPU)

Dla stylów ≥ sequential_scaled wymagany serwer z GPU NVIDIA (CUDA 12+). Dla sequential_light wystarczy CPU z 16GB RAM — wolniejsze, ale funkcjonalne.

Setup

Uruchomienie

Trzy kroki od zera do działającego serwisu MCP.

📄 .env (uzupełnij) 
# Opcjonalne — dla prywatnych modeli HF
HF_TOKEN=hf_...

# Wymagane dla stylu deliberation
TAVILY_API_KEY=tvly-...

# Styl i urządzenie
RECURSIVE_MAS_STYLE=sequential_light
RECURSIVE_MAS_DEVICE=cpu
🐳 docker compose 
docker compose -f docker-compose.mcp-tier.yml up recursive-mas
🔬 test inference 
curl -X POST http://localhost:8095/tools/recursive_mas_infer \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Explain latent space recursion"}'
